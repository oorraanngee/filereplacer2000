ПЕРЕЙТИ К ЗАГРУЗКЕ

File Replacer 2000

1. О ПРОГРАММЕ
Программное обеспечение предназначено для управления файлами проекта, редактирования исходного кода, проверки контрольных сумм и атрибутов доступа.

Разработано специально для быстрой и надежной работы на платформе Windows XP SP3 и выше.
Версия сборки: 0.5 (2010-2026) Теперь с новым .NET 4.0!
Прошлые версии устарели. Если у вас осталась копия программы 0.4 и ниже, то удалите и установите последнию вместе с .NET 4.0.
2. ИНТЕРФЕЙС
ПРОВОДНИК
Иерархическое дерево слева отображает структуру каталогов и файлов. При выборе объекта программа автоматически подготавливает его к работе согласно активному режиму.

АДРЕСНАЯ СТРОКА
Поле 'Адрес' предназначено для ввода абсолютного пути к директории или файлу. После ввода пути нажмите клавишу Enter для осуществления перехода.
3. ФАЙЛЫ И ПАПКИ
ДОСТУПНЫЕ ОПЕРАЦИИ (Вызываются контекстным меню по правой кнопке мыши или через панель инструментов):

• Создать файл — инициализация нового пустого файла.
• Создать папку — инициализация новой директории.
• Переименовать — изменение системного имени объекта.
• Удалить — физическое удаление объекта с накопителя.
• Обновить — принудительное перечитывание состояния файловой системы.
4. РЕЖИМ ПЕРЕЗАПИСИ
Данный режим предназначен для полной замены содержимого целевого файла.

Алгоритм работы:
1. Выберите целевой файл в обозревателе.
2. Внесите новый текст в текстовое поле справа.
3. Нажмите 'Заменить текст'. Текущее содержимое файла будет безвозвратно удалено и заменено введенными данными.
5. РЕЖИМ РЕДАКТОРА
Переключение режима осуществляется кнопкой на верхней панели. В данном режиме файл открывается для построчного редактирования.

Внимание: Видеофайлы запрещены для редактирования как текст, чтобы избежать зависаний. При открытии изображений автоматически активируется встроенный Paint для визуального редактирования.
6. МЕТАДАННЫЕ И ПРАВА
ОКНО СВОЙСТВ (Контекстное меню -> Метаданные, Хэш и Права)
- Временные атрибуты: Изменение системной даты создания, модификации и последнего открытия файла.
- Контрольные суммы: Вычисление алгоритмов MD5 и SHA-256 для верификации целостности файла, а также функция сравнения с эталонным значением.
- Списки контроля доступа (ACL): Просмотр разрешений. Иконки обозначают тип субъекта:
Человек (Пользователь),
Рука (Общий доступ),
Земля (Сеть),
Звезда (Администратор/Система).
7. РЕЗЕРВНЫЕ КОПИИ
РЕЗЕРВНОЕ КОПИРОВАНИЕ (.old)
При активации соответствующего параметра, перед записью изменений система создает дубликат оригинального файла в поддиректории old_ИмяФайла.

ВОССТАНОВЛЕНИЕ
При выборе файла с расширением .old система отображает дату его создания и активирует кнопки для создания нового файла из копии, сравнения (Diff) и восстановления.
Для работы на Windows XP SP3 и выше требуется наличие .NET Framework 4.0.
